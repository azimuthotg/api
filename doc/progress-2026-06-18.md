## บริบท / ปัญหา

ระบบผูกบัญชีเพื่อเปิดประตู RFID: นักศึกษา/บุคลากรเข้า LINE (ได้ UID) แล้วผูกกับ LDAP
(กรอก user/pass อินเทอร์เน็ต). มี **บางคนผูกไม่สำเร็จเพราะ authen ไม่ผ่าน** ทั้งที่ค้นใน DB
แล้วมีข้อมูล — แต่ระบบ **ไม่มี monitor/log เลย** เจ้าหน้าที่หน้างาน (บรรณารักษ์เคาน์เตอร์)
จึงไม่รู้ว่าเกิดอะไร และเวลาไปถามนักคอมก็ไม่มี error ให้ดู ("ถามนักคอมทีไรบอกไม่รู้ error")

**เป้าหมาย:** หน้า Monitor ให้บรรณารักษ์ดูกิจกรรมผูกบัญชี สำเร็จ/ไม่สำเร็จ พร้อม "สาเหตุ"
ที่อ่านเข้าใจ เพื่อนำไปแจ้งทีมคอมได้

## เพิ่มเติม (รอบบ่าย) — วินิจฉัยสาเหตุ AD ละเอียดขึ้น + หน้าทดสอบ login

### แยกสาเหตุจาก AD sub-code
- `check_ad_detailed()` เปลี่ยนจาก `auto_bind=True` (throw exception ทิ้ง result) เป็น `bind()`
  แล้วอ่าน `conn.result['message']` ดึง code `data XXX` ที่ AD ส่งกลับ (bind ครั้งเดียว ไม่เพิ่มเสี่ยงล็อก)
- แปล sub-code เป็นสาเหตุภาษาคน: `52e`=รหัสผิด/ไม่มีบัญชี, `525`=ไม่มี username, `532`=รหัสหมดอายุ,
  `533`=บัญชีถูกปิด, `701`=บัญชีหมดอายุ, `773`=ต้องเปลี่ยนรหัส, `775`=บัญชีถูกล็อก, `530/531`=AD ไม่อนุญาต
- เพิ่ม reason_code/label ใหม่ในหน้า Monitor ตามนี้
- **ข้อจำกัด:** AD ปกปิด "รหัสผิด" กับ "ไม่มี username" เป็น `52e` เหมือนกัน และ AD เช็ครหัสก่อนสถานะ
  → ด้วยรหัสที่ผิด จะตรวจได้แน่แค่ `775` (ล็อก); สถานะปิด/หมดอายุต้องมีรหัสถูกถึงจะเห็น
  → ไม่มี AD service account ในโปรเจกต์ใดเลย (ทุก endpoint รวม LibLogin bind ด้วยรหัส user)

### หน้าทดสอบ login กับ AD (`/monitor/adtest/`)
- `monitor_views.monitor_adtest` + `templates/monitor/adtest.html` (หลังรหัสร่วมเดียวกับ Monitor)
- เจ้าหน้าที่กรอก username+password → ยิงตรงไป AD → เห็นทันที ผ่าน/ไม่ผ่าน + สาเหตุ (+ ชื่อ/อีเมลถ้าผ่าน)
- ถ้านักศึกษากรอกรหัส**ถูก** แต่บัญชีมีปัญหา จะเห็น code จริง (533/532/773) ทะลุกำแพง 52e ได้
- **ไม่บันทึกลง BindingLog** (เป็นเครื่องมือทดสอบ) + มีคำเตือนเรื่องทดสอบรหัสผิดบ่อยอาจถูกล็อก
- มีปุ่มลิงก์จากหน้า Monitor

## หมายเหตุ / งานที่ยังเปิดอยู่
- ถ้า `displayName` ว่าง คอลัมน์ชื่อจะว่าง — แก้แล้วที่ฝั่ง reserv (LINE profile fallback)
- API **ไม่ยุ่งกับ LINE token เอง** โดยตั้งใจ (token อยู่ที่ reserv) — apiproject แค่รับ/เก็บ displayName
- **อยากตรวจสถานะบัญชี AD โดยไม่ต้องรู้รหัส** (มี/ไม่มีบัญชี, disabled, รหัสหมดอายุ) → ต้องขอ
  **read-only AD service account** จาก admin AD มาใส่ .env แล้วทำหน้าตรวจสถานะเพิ่ม (ยังไม่ได้ทำ)
- deploy: `.\deploy.ps1` (pull → migrate → collectstatic → recycle app pool `apiproject`)
