### API access log: แยกตารางสด/คลัง + retention (commits `7df72ef` `d175a2f` `55409c2` `99971a1`)
ปัญหา: หน้า `/monitor/api-usage/` query หนัก (COUNT/DISTINCT ทั้งตาราง) ทุกครั้งที่รีเฟรช 5 วิ + ตาราง `api_access_log` โตไม่มีเพดาน
แก้เป็น 2 ชั้น:
- ตารางสด `api_access_log` = เก็บเฉพาะ **วันนี้** → หน้า real-time อ่านจากนี่ เบาตลอด
- คลัง `api_access_archive` (model `ApiAccessArchive`, migration `0008`) = ดิบครบ เก็บ **90 วัน** ไว้วิเคราะห์
- command `rotate_access_log` (ย้ายแถวก่อนเที่ยงคืนไทยเข้าคลัง chunked 2000 + ลบคลังเกิน 90 วัน) + `rotate_access_log.ps1` + Windows Task Scheduler `NPUAPI rotate_access_log` รัน 00:10
- รันจริงครั้งแรกย้าย **5581 แถว** (สด 671 / คลัง 5581)
- หน้าใหม่ `/monitor/api-usage/analysis/` อ่านจากคลัง เลือกช่วงวันที่ (แยกจาก real-time ที่รีเฟรช)
- บทเรียน .ps1 (PowerShell 5.1): ต้อง `PYTHONIOENCODING=utf-8` + `[Console]::OutputEncoding=UTF8` (ไทยไม่เพี้ยน) และตัดสินสำเร็จ/พังจาก `$LASTEXITCODE` ไม่ใช่ stderr

### Timezone + date format (commits `86995eb` `fd9d32e` `f2d94dd` `675831b`)
- ลองตั้ง `TIME_ZONE=Asia/Bangkok` → **หน้า monitor ว่างเปล่าทันที!** เพราะ MySQL ยังไม่ได้โหลดตาราง timezone → Django สั่ง `CONVERT_TZ` ตอน filter `__date` แล้วคืน `NULL`
- แก้: คง `TIME_ZONE=UTC` (ไม่พึ่ง CONVERT_TZ) แล้วคิด "วันไทย" ที่ชั้นแอป — helper `_bkk_today`/`_bkk_day_range`, filter ด้วยช่วง `__gte/__lt` (ไม่ใช่ `__date`), แสดงเวลาไทยในเทมเพลตด้วย `|timezone:bkk_tz`
- วันที่ทุกหน้า monitor เป็น `d-m-Y` (dd-mm-yyyy)
- หน้า `/monitor/` อัปเกรด auto-refresh ให้เหมือน api-usage (ปุ่มเปิด/ปิด + ตัวนับ + จำค่า)
