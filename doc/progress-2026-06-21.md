## ที่มา
external member เดิมรองรับเฉพาะ **รหัสหมุนเวียนรายวัน** (`ExternalAccessViewSetV2.issue` + pool `ExternalAccessCode`)
เพิ่มประเภท **"ถาวร"** สำหรับคนภายนอกที่เข้าใช้ประจำ (เช่น พนักงานส่งเอกสาร) — admin อนุมัติครั้งเดียว
ได้รหัสคงที่ใช้ได้ทุกวันจนกว่าจะระงับ มีรูปสำหรับทำบัตร

## วิธีตรวจ end-to-end (หลัง migrate + reserv พร้อม)
1. reserv ลงทะเบียนถาวร → api สร้าง `pending`
2. admin approve ที่ reserv → api ออก `permanent_code` + `active`
3. `GET /v2/external/check/<permanent_code>/` → `allow:true` ทุกวัน (ทดสอบข้ามวัน)
4. revoke → check ต้อง `allow:false` ทันที
5. ดู `/monitor/api-usage/` ว่าทุก call ขึ้น log ระบบ=reserv
