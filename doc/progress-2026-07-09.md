# Progress 2026-07-09 — approve สมาชิกถาวร เก็บ approved_by จาก request

**เครื่องมือ:** Claude Code edit
**เกี่ยวข้อง:** reserv commit `344f6fc` (ฝั่ง client ส่ง field `approved_by` มาแล้ว)

---

## ที่มา

หน้า `/reserv/manage/external/<id>/` แสดง "อนุมัติเมื่อ ... โดย reserv" ทุกใบ เพราะ endpoint approve
เก็บ `approved_by = request.user.username` (= "reserv" ซึ่งเป็นบัญชี JWT ที่ reserv ใช้) ไม่ใช่ชื่อ
เจ้าหน้าที่จริงที่กดอนุมัติ

reserv แก้ให้ส่ง form field `approved_by` (ชื่อ staff ที่ล็อกอิน เช่น `admin_e`) มากับ request แล้ว
ฝั่ง api จึงต้องอ่านค่านี้ไปเก็บ

## สิ่งที่แก้

`apiapp/views_v2.py` — `ExternalAccessViewSetV2.permanent_approve()` (1 จุด):
```python
# เดิม
member.approved_by = getattr(request.user, 'username', None)
# ใหม่
approved_by = (request.data.get('approved_by') or '').strip()[:150]
member.approved_by = approved_by or getattr(request.user, 'username', None)
```
- อ่าน `approved_by` จาก request ถ้ามี → เก็บค่านั้น (cap 150 ตาม max_length ของฟิลด์)
- ถ้าไม่มี/ว่าง → fallback เป็น username ของ JWT เหมือนเดิม (ไม่พังของเก่า/ client อื่น)
- ไม่แตะ revoke / permanent_code / status / serializer (serializer return `approved_by` อยู่แล้ว)
